# بوت تيليغرام — أخبار الصحة اليومية بالعربية

بوت يرسل إليك كل يوم على تيليغرام آخر المقالات الصحية باللغة العربية،
مجمّعة من عدة مصادر RSS عربية (BBC عربي، الجزيرة، منظمة الصحة العالمية، وِبطِب، اليوم السابع، صحة نيوز).

يعمل **بدون خادم** بفضل GitHub Actions: بعد الإعداد مرة واحدة، يُشغَّل تلقائيًا في الوقت المحدد. مجاني.

---

## نظرة عامة على الآلية

```
GitHub Actions (جدول يومي)
        │
        ▼
   health_bot.py  ──►  يقرأ مصادر RSS  ──►  يصفّي آخر 24 ساعة
        │
        ▼
   API Telegram  ──►  رسالة على محادثتك
```

---

## الإعداد خطوة بخطوة

### 1. إنشاء بوت تيليغرام

1. على تيليغرام، افتح محادثة مع **@BotFather**.
2. أرسل `/newbot`، اختر اسمًا ومعرفًا (`...bot`).
3. سيعطيك BotFather **رمزًا (token)** من الشكل `123456789:AAE...`. احتفظ به.

### 2. الحصول على `chat_id`

1. أرسل أي رسالة إلى بوتك الجديد (اضغط على الرابط الذي أعطاه BotFather، ثم "ابدأ").
2. محليًا، شغّل:
   ```bash
   pip install -r requirements.txt
   export TELEGRAM_TOKEN="رمزك_هنا"
   python health_bot.py getid
   ```
3. يعرض السكريبت `TELEGRAM_CHAT_ID` الخاص بك. احتفظ به.

> لاستقبال الأخبار في **مجموعة** أو **قناة**: أضف البوت كعضو (مشرف للقناة)، أرسل رسالة، ثم أعد تشغيل `getid`. معرّف المجموعة/القناة يبدأ عادةً بـ `-100`.

### 3. اختبار محلي (اختياري لكن مُوصى به)

```bash
export TELEGRAM_TOKEN="رمزك_هنا"
export TELEGRAM_CHAT_ID="معرفك_هنا"
python health_bot.py
```
يجب أن تستلم الرسالة على تيليغرام في ثوانٍ.

### 4. النشر على GitHub (تشغيل تلقائي)

1. أنشئ مستودعًا على GitHub (يمكن أن يكون **خاصًا**) وارفع هذه الملفات:
   ```
   health_bot.py
   requirements.txt
   .github/workflows/daily-health-news.yml
   ```
   ⚠️ ضع ملف `.yml` داخل مجلد `.github/workflows/`.

2. في المستودع: **Settings → Secrets and variables → Actions → New repository secret**.
   أضف سرّين:
   - `TELEGRAM_TOKEN` → رمزك
   - `TELEGRAM_CHAT_ID` → معرفك

3. تبويب **Actions**: فعّل الـ workflows إذا طُلب ذلك. اختبر يدويًا بـ **Run workflow**.

هذا كل شيء. بعد ذلك سيُشغَّل البوت كل يوم في وقت الـ cron المحدد.

## الإعدادات الشائعة

| ماذا | أين | كيف |
|------|-----|-----|
| **وقت الإرسال** | `daily-health-news.yml`، سطر `cron` | بتوقيت UTC. `0 6 * * *` = 7 صباحًا بتوقيت المغرب. لـ 8 صباحًا → `0 7 * * *`. |
| **المصادر** | `health_bot.py`، قائمة `FEEDS` | أضف/احذف أزواجًا من نوع `("الاسم", "رابط_rss")`. |
| **عدد المقالات** | سر/متغير `MAX_ARTICLES` | الافتراضي 12. |
| **نافذة الحداثة** | متغير `MAX_AGE_HOURS` | الافتراضي 28 ساعة. |

أي مصدر RSS يتعطّل يُتجاهَل تلقائيًا: البوت يواصل مع المصادر الأخرى ولا يتوقف.

## أفكار للتطوير

- **ملخص بالذكاء الاصطناعي**: تلخيص كل مقال في جملة واحدة عبر Anthropic API قبل الإرسال.
- **تصفية بالكلمات المفتاحية**: الاحتفاظ فقط بالمقالات التي تحتوي على موضوعات معينة (تغذية، رياضة، مايكروبيوم...).
- **مصادر إضافية**: إضافة مصادر عربية أخرى إلى قائمة `FEEDS`.
- **إرسالات متعددة يوميًا**: إضافة أسطر `cron` إضافية.
